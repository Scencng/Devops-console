# 局限与后续规划

## 当前局限

- 当前连接是浏览器会话级连接，还不是多连接持久管理
- 还没有用户登录和平台级权限控制
- database 创建 / 删除还没有做成稳定 UI 能力
- 条件查询目前是单层条件组，不支持复杂嵌套分组
- 表单编辑优先支持简单字段，复杂嵌套结构更适合 JSON 模式
- 目前只做到 collection 级备份恢复，没有整 database 级备份恢复
- 对于全新 MongoDB 实例，通常仍需要先做一次命令行初始化

## 后续规划

### 第一阶段：继续提升可用性与初始化体验

- 继续优化连接页和错误提示
- 补充 database 创建 / 删除能力
- 优化空实例引导
- 进一步提升查询构建器体验
- 优化表单编辑模式
- 增强导入导出和备份恢复的可理解性

### 第二阶段：补充更贴近 MongoDB 的能力

- 继续完善索引查看与基础索引管理
- 支持更复杂的查询构建器
- 提供更清晰的 collection 统计信息

### 第三阶段：按需要扩展平台化能力

- 支持多 MongoDB 连接管理
- 支持 database 级备份恢复
- 支持聚合管道可视化
- 支持更完整的权限管理和审计
- 评估是否需要为部署环境补充更细粒度的 RBAC

## 当前明确不优先做的方向

为了保证项目聚焦，这些方向暂时不优先：

- 多连接管理
- 企业级权限体系
- DBA 平台化治理
- 复杂聚合可视化构建器
- 整 database 级恢复
